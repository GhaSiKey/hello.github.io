💡 修改建议
🔴 高优先级: BLRouter.performCreateModules 异步化
BLRouter.performCreateModules() 在主线程同步执行，包含网络IO和文件IO操作。改为异步执行可解决 Unsafe.park ANR。
// 当前（阻塞主线程） if (BiliContext.isMainProcess) BLRouter.performCreateModules() // 建议（异步执行） if (BiliContext.isMainProcess) { GlobalScope.launch(Dispatchers.IO) { BLRouter.performCreateModules() } } // 注意：需要评估是否影响依赖此模块的功能
🟡 中优先级: NeuronRemoteService 异步初始化
NeuronRemoteService.onCreate 使用 runBlocking 同步执行初始化，导致 ANR。改为异步初始化。

⚠️ 这些问题的堆栈多为 "Main thread idle"
Google 官方说明：当 ANR 条件已经过去时才捕获堆栈，导致堆栈中主线程处于空闲状态。 这类 ANR 通常由系统高负载或资源竞争导致，无法从堆栈直接定位根因。

间接原因：当 Unsafe.park 问题导致主线程阻塞时，系统负载升高，可能导致其他正常的应用操作也触发 ANR。

建议：优先解决 Unsafe.park 问题（类别1），这类 nativePollOnce 的 ANR 数量可能会自然下降。
💡 修改建议
🔴 高优先级: 解决 Unsafe.park 问题（间接解决本类别问题）
当主线程不再被 BLRouter.performCreateModules 阻塞后，系统的输入分发能力会恢复正常，Input dispatching ANR 会大幅减少。
🟡 中优先级: BroadcastReceiver 中避免阻塞IO
FirebaseInstanceIdReceiver 回调中执行了文件IO操作，应该移到后台线程。
🟡 中优先级: Facebook/Google Ads SDK 延迟初始化
AdvertisingIdClient 和 FacebookSdk 在启动时同步等待网络响应，建议延迟到首次使用时。
